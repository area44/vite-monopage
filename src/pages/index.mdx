## Inline Elements

Markdown inline formatting is styled consistently throughout your content.

- **Bold** highlights important information.
- *Italic* adds emphasis to text.
- `Inline code` is ideal for commands, filenames, and variables.
- **Combined _formatting_** works naturally.
- [Hyperlinks](https://en.wikipedia.org/wiki/Hyperlink) inherit the theme and include accessible hover and focus states.

## Lists

1. **Prerequisites & Setup**
   - Ensure [Node.js](https://nodejs.org/en) is installed on your local development machine.
   - Install the project package manager globally:
     1. Run `npm install -g pnpm`
     2. Verify the installation via `pnpm --version`
2. **Local Environment Deployment**
   - Clone the source repository to your workspace.
   - Run initial package installation:
     ```bash
     pnpm install
     ```
   - Launch the modern development server:
     - Use `pnpm dev` for local hot-reloads.
     - View the application live at `http://localhost:5173`.
3. **Production Distribution**
   - Run the compiler build task:
     ```bash
     pnpm build
     ```

## Math

Both inline and display equations are rendered using KaTeX.

- **Inline Math**: You can write equations inline like $E = mc^2$ or the quadratic formula $x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$ within continuous prose.
- **Display Math**: For complex equations, use display block rendering. Here is Maxwell's equations as an example:

$$
\nabla \cdot \mathbf{E} = \frac{\rho}{\varepsilon_0}
$$
